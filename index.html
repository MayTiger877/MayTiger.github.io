Deep Learning
De-Reverberation Tool

A neural audio enhancement system that removes reverberation and background noise from speech signals using a dual U-Net architecture trained end-to-end on spectrograms.

View on GitHub →

Overview

When speech is recorded in real environments, two kinds of degradation pile up simultaneously: reverberation (acoustic reflections from walls and surfaces that smear and blur the signal) and background noise (ambient sound layered on top). Both destroy speech intelligibility and make downstream tasks like speech recognition or speaker identification fail.

This project tackles both problems in a single trained pipeline by combining two specialist neural networks, one dedicated to each artifact, and chaining them in the order that empirically works best.

My Contribution

Built on top of the open-source Neural-Speech-Dereverberation codebase by Leon and Tobar. My work focused on:

⚙

Pipeline and Data Handling

Redesigned the training orchestrator and data pipeline to support dual-model chaining, custom dataset loading, and SLURM-based HPC batch submission.

🧠

Model Modifications

Modified individual functions within the LS-UNET (dereverberation) and standard U-Net (denoising) architectures to accept chained spectrogram inputs and share a common output format.

🔬

Hyperparameter Tuning

Ran multiple rounds of hyperparameter search across learning rate, batch size, and model depth until results were satisfying both by MSE metric and by ear.

📊

Evaluation Suite

Integrated five objective speech quality metrics to quantitatively evaluate models at every stage of the pipeline across pre-training and fine-tuning.

Architecture

The system runs two networks in sequence, operating on mel-spectrograms of the input signal. The frequency range is focused on 20 to 8300 Hz, the range relevant for speech intelligibility.

Evaluation Metrics

Five objective metrics were tracked throughout all stages to measure enhancement quality beyond just MSE loss:

STOI
Short-Term Objective Intelligibility
Measures how intelligible enhanced speech is to a human listener. Correlates well with subjective listening tests.
CD
Cepstral Distance
Distance between cepstral coefficients of clean and enhanced speech. Lower is better.
LLR
Log-Likelihood Ratio
Spectral distortion measure based on LPC analysis. Sensitive to spectral shape changes.
fwSNRseg
Frequency-Weighted Segmental SNR
SNR computed per segment and weighted by frequency importance to human hearing.
SRMR
Speech-to-Reverberation Modulation Energy Ratio
Directly measures reverberation level. Primary indicator of dereverberation quality.

BallPit

A physics-based MIDI generator where bouncing balls trigger notes. Each collision with a wall or another ball fires a customizable MIDI event, controlled chaos by design.

View on GitHub →

Overview

BallPit is a MIDI plugin inspired by the unpredictable, chaotic energy of a real ball pit. Balls bounce around a 2D pit and generate MIDI notes every time they collide with the pit walls or with each other. The scale, key, rhythm, and behavior of every ball is fully customizable, making it a generative composition tool as much as it is a performance instrument.

Built entirely in C++ using the JUCE framework, BallPit ships as a VST3 plugin with a CI/CD pipeline on GitHub Actions for automated builds on both Windows and Mac.

Showcase

Interface

Features

🎵

Scale and Key Control

Set the root note, key/scale, and note range. Notes are spread across the pit edges and triggered on collision.

⚙

Two Movement Modes

Chaos mode: freely set speed and direction per ball. Tempo mode: speed synced to DAW tempo on both X and Y axes independently.

🎲

Probability and Variation

Per-note probability controls the odds each note fires. Volume variation adds organic randomness to note velocity.

🔁

Per-Ball Delay Tail

Each ball has its own delay section with repeats, feedback, timing, and note transposition, turning one collision into a rhythm.

📐

Snap to Grid

Option to snap balls to a 1/8-interval grid on the pit, giving tighter rhythmic control over starting positions.

🎻

Quantization

Global quantization with adjustable size and percentage. Dial in how strictly notes align to the grid.

Starting X/Y coordinates of the ball on the pit.

Exodus Delay

A delay plugin where every repeat has its own character. Each delay line instance runs its own independent chain of effects, Gain, Pan, Distortion, Phaser, and Reverb, making every echo unique.

View on GitHub →

Overview

Most delay plugins treat every repeat the same: the signal feeds back at a fixed level and decays uniformly. Exodus Delay breaks that model. Each delay repeat is its own instance with its own FX chain. You can have the first repeat dry, the second distorted and panned left, the third filtered through a phaser, and the fourth drowned in reverb.

The result is a delay that evolves, rhythmically and texturally, across its repeats. Built in C++ with JUCE, targeting VST3 and AU formats with a CI/CD pipeline for automated cross-platform builds.

Interface

Features

🔗

Per-Repeat FX Chain

Each delay line instance has its own independent effects chain: Gain, Pan, Distortion, Phaser, and Reverb, all fully customizable.

🎢

Gain and Pan

Control the volume and stereo position of each individual repeat independently from the others.

⚡

Distortion

Per-repeat waveshaping distortion. Add grit and harmonic saturation to specific echoes while keeping others clean.

🌀

Phaser

Phase modulation on individual delay lines for sweeping, swirling echoes with their own movement and character.

🌊

Reverb

Apply reverb per delay repeat. Send specific echoes into space while keeping others tight and dry.

⏱

General Delay Settings

Full control over global delay time, feedback, wet/dry mix, and sync to DAW tempo.

How It Works

Exodus routes audio through a sequencer of delay taps. Each tap reads from the delay buffer at its designated time offset and passes its output through its own DSP chain before being summed into the output. This means the plugin is not just a standard feedback delay. It is closer to a programmable multi-tap delay with per-tap signal processing.
